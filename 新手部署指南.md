## 第一步：准备Windy API Key

### 1.1 注册Windy账户
1. 打开浏览器，访问 https://api.windy.com/
2. 点击右上角 "Sign Up" 注册
3. 填写邮箱、密码，完成注册
4. 验证邮箱（检查邮件点击验证链接）

### 1.2 申请API Key
1. 登录后，点击 "Get API Key"
2. 选择 "Free Plan" (1000次/月免费)
3. 填写应用信息：
   - App Name: `Surf Forecast Light`
   - Description: `Personal surf forecast website`
   - Website: `https://your-username.github.io/surf-forecast`
4. 点击 "Create API Key"
5. **重要**: 复制并保存API Key（类似：`abc123def456...`）

---

## 第二步：上传代码到GitHub

### 2.1 创建GitHub仓库
1. 登录 https://github.com
2. 点击右上角 "+" → "New repository"
3. 填写仓库信息：
   - Repository name: `surf-forecast-light`
   - Description: `AI-powered surf forecast website`
   - 选择 "Public"
   - 勾选 "Add a README file"
4. 点击 "Create repository"

### 2.2 上传项目文件
**方法A: 网页上传（推荐新手）**
1. 在仓库页面点击 "uploading an existing file"
2. 将整个 `C:\surf recommendation\light版\` 文件夹拖拽到页面
3. 等待上传完成
4. 在底部填写：
   - Commit message: `Initial commit - surf forecast website`
5. 点击 "Commit changes"

**方法B: Git命令行**
```bash
# 打开命令提示符，进入项目目录
cd "C:\surf recommendation\light版"

# 初始化Git仓库
git init

# 添加远程仓库（替换your-username）
git remote add origin https://github.com/your-username/surf-forecast-light.git

# 添加所有文件
git add .

# 提交
git commit -m "Initial commit - surf forecast website"

# 推送到GitHub
git push -u origin main
```

## 第三步：部署到Vercel

### 3.1 注册Vercel账户
1. 访问 https://vercel.com
2. 点击 "Sign Up"
3. 选择 "Continue with GitHub" 
4. 授权Vercel访问GitHub

### 3.2 导入项目
1. 在Vercel控制台点击 "New Project"
2. 找到你的 `surf-forecast-light` 仓库
3. 点击 "Import"
4. 保持默认设置，点击 "Deploy"
5. 等待部署完成（约1-2分钟）

### 3.3 设置API Key环境变量
1. 部署完成后，点击 "Go to Dashboard"
2. 在项目页面点击 "Settings" 标签
3. 点击左侧 "Environment Variables"
4. 点击 "Add New"
5. 填写：
   - Name: `WINDY_API_KEY`
   - Value: `你的Windy API Key`
   - Environment: 选择 "Production"
6. 点击 "Save"

### 3.4 重新部署
1. 回到 "Deployments" 标签
2. 点击最新部署右侧的 "..." 
3. 选择 "Redeploy"
4. 等待重新部署完成

---

## 第四步：测试网站

### 4.1 访问网站
1. 在Vercel项目页面找到域名（如：`surf-forecast-light.vercel.app`）
2. 点击访问网站
3. 检查是否正常显示浪点数据

### 4.2 验证API工作
1. 打开浏览器开发者工具（F12）
2. 切换到 "Network" 标签
3. 刷新网站
4. 查看是否有 `windy-proxy` 请求
5. 检查请求状态是否为 200

### 4.3 检查数据来源
**方法1: 在网站页面直接查看**
- 在每个浪点预测卡片的底部，会显示"数据源"信息
- 真实数据显示："数据源: 实时"
- 备用数据显示："数据源: 模拟"

**方法2: 在开发者工具中查看**
1. 按F12打开开发者工具
2. 切换到"Network"标签
3. 刷新网站
4. 点击`windy-proxy`请求
5. 查看Response响应数据中的`source`字段：
   - `"source": "windy"` = 真实Windy数据
   - `"source": "fallback"` = 备用模拟数据

---

## 第五步：配置自定义域名（可选）

### 5.1 购买域名
- 推荐：Namecheap, GoDaddy, 阿里云等
- 建议域名：`surfforecast.com`, `wavespot.net` 等

### 5.2 在Vercel绑定域名
1. 在Vercel项目设置中点击 "Domains"
2. 输入你的域名，点击 "Add"
3. 按照提示配置DNS记录
4. 等待DNS生效（通常几分钟到几小时）

## 🔧 常见问题解决

### Q1: API调用失败
**症状**: 网站显示"数据源: 模拟"，控制台有错误
**解决**:
1. 检查Vercel环境变量是否正确设置
2. 确认API Key有效且未过期
3. 检查API调用次数是否超限

### Q2: 网站无法访问
**症状**: 打开Vercel域名显示404或错误
**解决**:
1. 确认GitHub仓库文件上传完整
2. 检查Vercel部署日志是否有错误
3. 尝试重新部署

### Q3: 数据不更新
**症状**: 浪况数据始终相同
**解决**:
1. 数据有6小时缓存，属正常现象
2. 可以清除浏览器缓存强制刷新
3. 检查API调用是否正常

### Q4: 部署失败
**症状**: Vercel部署过程中报错
**解决**:
1. 检查代码文件是否完整
2. 确认没有语法错误
3. 查看部署日志具体错误信息

### Q5: API调用排查工具
**使用API测试页面**:
1. 访问 `https://your-project.vercel.app/api-test.html`
2. 依次点击所有测试按钮
3. 查看详细的错误信息和调试日志
4. 将结果截图发给技术支持
